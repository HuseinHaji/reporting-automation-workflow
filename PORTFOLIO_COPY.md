# Portfolio Copy

## Short Card Description

Automated a synthetic monthly reporting workflow using Python, SQL, validation checks, exception reporting, and BI-ready exports.

## Long Project Description

Reporting Automation Workflow is a portfolio implementation that simulates a recurring business reporting process for a B2B financial-services style dataset. The project starts with synthetic raw files for customers, invoices, payments, contracts, cases, and monthly targets, then automates validation, transformation, KPI calculation, exception detection, and dashboard-ready export creation.

The workflow is designed to demonstrate how a BI analyst or reporting analyst can reduce repetitive manual reporting steps while improving consistency and transparency. It includes a data quality audit, business-readable exception report, monthly KPI summary, customer reporting table, PostgreSQL-style SQL examples, pytest coverage, Docker support, GitHub Actions, and an optional Streamlit reporting demo.

The project does not use real company data and does not claim real business impact. It is designed to demonstrate practical analyst thinking: turning raw synthetic business data into validated reports, exception logs, KPI summaries, and BI-ready outputs.

## Skills List

Python, pandas, SQL, data validation, KPI calculation, exception reporting, reporting automation, Streamlit, Power BI-ready modeling, Docker, CI/CD, pytest.
